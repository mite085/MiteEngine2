# 待确认问题

**用途**：临时记录需要人类确认的问题，协调者继续推进任务

---

## 使用规则

```
1. 协调者遇到需要人类确认的问题时
2. 记录到本文件夹（每个问题一个文件）
3. 协调者继续推进，不阻塞
4. 人类确认后，更新到迭代文档
5. 删除本文件夹中的问题文件
```

---

## 问题模板

```markdown
# 待确认问题

**编号**：001
**创建时间**：2026-03-09 HH:MM
**状态**：待确认 / 已确认 / 已解决

## 问题描述

[简要描述问题]

## 上下文

[相关的代码/设计/决策]

## 选项

- 选项A：[描述]
- 选项B：[描述]

## 协调者建议

[建议的选项及原因]

---

**确认后**：更新到 `docs/协调者迭代/协调者001-架构设计阶段.md`，然后删除本文件
```

---

## 示例

```
docs/
└── 待确认问题/
    ├── 001-SDF渲染器实现选择.md
    └── 002-模块划分调整.md
```

---

## 处理流程

```
发现问题
    ↓
记录到 docs/待确认问题/XXX.md
    ↓
继续推进任务
    ↓
人类查看并确认
    ↓
更新到迭代文档
    ↓
删除待确认问题文件
```
