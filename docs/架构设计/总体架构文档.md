## 1. 模块划分方案

### 1.1 模块总览

基于需求文档中的100万行代码目标和核心特性，将引擎划分为以下七大模块：

```
┌─────────────────────────────────────────────────────────────────┐
│                        Editor (编辑器)                           │
│                    ~15万行 | 独立可执行进程                      │
├─────────────────────────────────────────────────────────────────┤
│                     Game Runtime (游戏运行时)                    │
│  ┌─────────────┬─────────────┬─────────────┬─────────────────┐  │
│  │  Rendering  │   Physics   │    ECS      │   Resources    │  │
│  │  ~25万行    │  ~15万行    │  ~10万行    │    ~10万行     │  │
│  ├─────────────┴─────────────┴─────────────┴─────────────────┤  │
│  │                     Core (核心层) ~30万行                   │  │
│  └───────────────────────────────────────────────────────────┘  │
├─────────────────────────────────────────────────────────────────┤
│                    Platform (平台抽象层)                          │
│                    ~5万行 | 跨平台支持                            │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 各模块职责定义

| 模块 | 职责 | 关键类/子系统 |
|------|------|--------------|
| **Core** | 基础架构、内存管理、数学库、基础工具 | Memory, Math, Logger, String, Thread, FileSystem |
| **Rendering** | 图形渲染、管线管理、材质系统 | RHI, RenderGraph, SDFRenderer, PostProcess |
| **Physics** | 碰撞检测、相位物理、空间查询 | Collision, SDFCollision, PhasePhysics |
| **ECS** | 实体管理、组件系统、系统调度 | Entity, Component, System, World |
| **Resources** | 资源加载、缓存、序列化 | Asset, AssetManager, Serializer |
| **Editor** | 场景编辑、SDF编辑、热重载、UI | EditorScene, SDFEditor, HotReload, ImGuiExt |
| **Platform** | 窗口管理、输入、设备抽象 | Window, Input, Device |

**每层规则**：
- 只能向下依赖，不能反向调用
- 同层之间通过接口通信
- 依赖注入替代直接依赖

### 3.2 第三方库引入策略

**核心原则**：积极引入成熟开源库，避免重复造轮子

| 评估维度 | 标准 | 推荐库 |
|---------|------|--------|
| **数学库** | 价值高（实现成本高、SIMD优化复杂）、风险低（稳定、广泛使用） | GLM（推荐，与GLSL语义兼容）或 Eigen（高性能） |
| **线程库** | 价值高（跨平台同步原语复杂）、风险低（标准库/Boost成熟） | std::thread（C++20）或 Boost.Thread |
| **ECS库** | EnTT 已选定 | EnTT 3.12+（Header-only，高性能） |
| **UI框架** | ImGui 已选定 | ImGui（游戏UI标准选择） |
| **窗口管理** | GLFW 已选定 | GLFW 3.3+ |

**决策流程**：
1. 评估引入价值：是否能显著降低开发时间成本
2. 评估风险：库是否稳定、社区是否活跃、许可证是否兼容
3. 优先选择 Header-only 库，减少构建复杂度

#### 4.1.3 Raymarching着色器核心

```cpp
// GLSL 核心算法伪代码
float SceneSDF(vec3 p) {
    float d = INF;

    // 遍历所有SDF物体
    for (auto& obj : scene.objects) {
        // 相位剔除：只渲染当前相位
        if (obj.phaseMask & currentPhase) {
            d = min(d, obj.sdf->Evaluate(p));
        }
    }

    return d;
}

vec3 CalculateNormal(vec3 p) {
    const float eps = 0.001;
    return normalize(vec3(
        SceneSDF(p + vec3(eps, 0, 0)) - SceneSDF(p - vec3(eps, 0, 0)),
        SceneSDF(p + vec3(0, eps, 0)) - SceneSDF(p - vec3(0, eps, 0)),
        SceneSDF(p + vec3(0, 0, eps)) - SceneSDF(p - vec3(0, 0, eps))
    ));
}

float RayMarch(vec3 ro, vec3 rd, float maxDist) {
    float t = 0;
    for (int i = 0; i < MAX_STEPS; i++) {
        vec3 p = ro + rd * t;
        float d = SceneSDF(p);
        if (d < SURFACE_DIST) return t;
        if (t > maxDist) break;
        t += d;
    }
    return -1;
}
```

```cpp
    void Update(float deltaTime) override;

    // 玩家切换相位

    void ApplyPhaseTransition(Entity entity, Phase from, Phase to);
    void UpdatePhaseVisuals(Entity entity, Phase phase);
    void HandlePhaseTrigger(Entity entity, Phase newPhase);

// 碰撞过滤逻辑
bool PhaseSystem::ShouldCollide(Entity a, Entity b) const {
    auto* phaseA = world.GetComponent<PhaseComponent>(a);
    auto* phaseB = world.GetComponent<PhaseComponent>(b);

    if (!phaseA || !phaseB) return true;

    // 两者必须在至少一个共同相位下都有碰撞
    PhaseMask collisionMask = phaseA->collisionPhases & phaseB->collisionPhases;
    return collisionMask != 0;
}
```

#### 4.3.3 SDF精确碰撞

```cpp
// 精确碰撞核心算法
class SDFCollisionDetector {
public:
    // SDF与射线的精确碰撞
    static bool SDFRayCast(const SDFNode& sdf, const Vec3& origin,
                          const Vec3& direction, float maxDistance,
                          RayHit& hit) {
        float t = 0;
        Vec3 p = origin;

        for (int i = 0; i < MAX_STEPS; i++) {
            float d = sdf.Evaluate(p);

            if (d < SURFACE_THRESHOLD) {
                // 击中表面
                hit.t = t;
                hit.position = p;
                hit.normal = sdf.CalculateNormal(p);
                return true;
            }

            t += d;
            p = origin + direction * t;

            if (t > maxDistance) break;
        }

        return false;
    }

    // SDF与球体的精确碰撞（用于物理模拟）
    static bool SDFSphereCollision(const SDFNode& sdf, const Vec3& sphereCenter,
                                   float radius, Contact& contact) {
        float d = sdf.Evaluate(sphereCenter);

        if (d < radius) {
            // 穿透：计算穿透深度和法线
            Vec3 normal = sdf.CalculateNormal(sphereCenter);
            contact.penetrationDepth = radius - d;
            contact.normal = normal;
            contact.position = sphereCenter - normal * d;
            return true;
        }

        return false;
    }
};
```

```cpp
// 碰撞查询时考虑相位
class PhaseCollisionFilter {
public:
    static bool CanCollide(const PhaseComponent& a, const PhaseComponent& b,
                          Phase currentPhase) {
        // 只有当两者在当前相位都有碰撞体时才碰撞
        bool aHasCollision = (a.collisionPhases & (1 << currentPhase.id)) != 0;
        bool bHasCollision = (b.collisionPhases & (1 << currentPhase.id)) != 0;

        return aHasCollision && bHasCollision;
    }
};

// 玩家控制器使用示例
void PlayerController::Move(const Vec3& displacement) {
    Vec3 newPos = transform_->position + displacement;

    // 检查SDF碰撞
    for (auto& obstacle : scene.GetObstacles()) {
        auto* phaseComp = world.GetComponent<PhaseComponent>(obstacle);

        // 相位过滤
        if (!PhaseCollisionFilter::CanCollide(playerPhase_, *phaseComp,
                                              currentPhase_)) {
            continue;  // 跳过不同相位的物体
        }

        SDFCollisionShape collider(obstacle.sdfRoot, obstacle.phaseMask);

        if (collider.SphereCast(newPos, playerRadius_, normal, depth)) {
            // 处理碰撞响应
            newPos += normal * depth;
        }
    }

    transform_->position = newPos;
}
```
