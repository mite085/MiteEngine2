## 1. 团队角色定义

| 角色 | 类型 | 核心职责 | 输出产物 |
|------|------|---------|---------|
| **协调者** | 人类+AI | 任务分发、进度管理、结果汇总、状态维护 | 迭代文档更新 |
| **总架构师** | Plan Agent | 整体架构规划、模块划分、技术选型 | 总体架构文档 |
| **接口人** | Plan Agent | 模块间接口协调、解耦审查、整体一致性 | 接口规范文档 |
| **模块架构师** | Plan Agent | 各模块的详细架构设计 | 模块架构文档（可多个） |
| **开发规范制定者** | Plan Agent | 代码/PR/文档规范制定 | 规范文档、配置文件 |
| **项目经理** | General Agent | 进度跟踪、日常协调、资源分配 | 进度报告、任务状态 |
| **开发者** | General Agent | 具体功能实现、代码编写 | 可编译代码 |
| **测试** | General Agent | 测试用例生成、验证正确性 | 测试代码/报告 |
| **Reviewer** | General Agent | 对抗式代码审查、质量把关、问题发现 | 审查报告 |
| **Approver** | General Agent | PR/MR 审核、合并决策、质量复核 | 审核报告 |

## 2. Rules（规则）

### R1. 任务分配规则

```
规则说明：定义什么任务分配给什么角色

分配原则：
- 架构设计、技术选型 → 架构师 Agent
- 具体功能实现、代码编写 → 开发者 Agent
- 测试用例生成、逻辑验证 → 测试 Agent
- 代码质量审查、问题发现 → Reviewer Agent
- 任务分解、进度规划 → 协调者（我）

特殊情况：
- 小任务（<100行代码）可由开发者直接完成
- 复杂任务需先经架构师设计
- Reviewer 发现问题 → 退回开发者修复 → 重新 Review
```

### R2. 上下文继承规则

```
规则说明：每次任务交接时，下一个 Agent 必须接收完整上下文

必须包含的信息：
1. 上一个 Agent 的完整输出
2. 相关需求文档引用（需求迭代文档.md 等）
3. 已有的设计约束和决策记录
4. 当前任务的目标和验收标准

继承流程：
开发者 ← 架构师：接收架构设计文档
测试 ← 开发者：接收代码实现
Reviewer ← 测试：接收测试用例 + 源码
```

### R3. 阻塞上报规则

```
规则说明：Agent 遇到无法解决的问题时必须上报

上报条件：
- 需求不明确，无法继续
- 技术方案不可行
- 遇到未知错误且无法解决
- 需要人工决策

上报内容：
- 问题描述
- 已尝试的解决方案
- 可能的替代方案
- 建议

处理流程：
阻塞 → 记录到迭代文档 → 等待协调者/人类决策
```

### R4. 质量门禁规则

```
规则说明：每个角色的输出必须满足最低质量标准

质量标准：
- 架构师：输出架构设计文档，包含模块划分、接口定义
- 开发者：代码可编译通过，遵循项目编码规范
- 测试：测试用例覆盖核心逻辑，有明确的通过/失败标准
- Reviewer：给出明确结论（通过/需修改），列出具体问题

门禁流程：
任务完成 → 质量检查 → 不通过则退回 → 重新执行
```

### R5. 决策追溯规则

```
规则说明：所有关键决策必须记录在案

必须记录的内容：
- 决策内容（what）
- 决策原因（why）
- 决策者
- 决策时间
- 反对意见（如有）

记录位置：迭代文档中的「历史决策」章节
```

## 3. Skills（技能）

### S1. 任务规划能力

```
技能名称：task_planning

用途：把一个大任务分解成可执行的子任务

使用方法：启动 Plan Agent

输入：高层任务描述、需求文档
输出：子任务列表、依赖关系、执行顺序
```

### S2. 代码审查能力

```
技能名称：code_review

用途：对抗式审查，确保代码质量

使用方法：启动 General Agent + Grep/Read 工具

审查维度：
- 代码规范
- 潜在bug
- 性能问题
- 安全问题
- 设计合理性

输出：审查报告，包含问题列表和严重程度
```

### S3. 探索研究能力

```
技能名称：codebase_explore

用途：调研技术方案、评估可行性、理解现有代码

使用方法：启动 Explore Agent

能力：
- 按模式搜索文件
- 理解代码结构
- 回答技术问题

输出：调研报告、技术分析
```

### S4. 测试生成能力

```
技能名称：test_generation

用途：根据需求生成测试用例

使用方法：启动 General Agent + Write 工具

测试类型：
- 单元测试
- 集成测试
- 回归测试

输出：测试代码文件
```

### S5. 代码生成能力

```
技能名称：code_generation

用途：具体实现功能

使用方法：启动 General Agent + Write/Edit 工具

能力：
- 根据设计文档实现功能
- 遵循编码规范
- 生成可编译代码

输出：源代码文件
```

## 4. 协作流程

### 4.1 标准工作流程

```
1. 协调者接收人类任务
       ↓
2. 启动架构师 Agent → 设计架构
       ↓
3. 架构师产出《第三方库评审清单》 → 人类逐项确认
       ↓
4. 启动开发者 Agent → PoC骨架验证 + 接口编译验证
       ↓
5. 人类确认架构可落地
       ↓
6. 启动开发者 Agent → 实现功能
       ↓
7. 启动测试 Agent → 生成测试
       ↓
8. 启动 Reviewer Agent → 代码审查
       ↓
9. 汇总结果 → 更新迭代文档 → 自动推进下一任务
```

### 4.2 迭代模式

```
协调者自动推进：

检查迭代文档：
- 有待办任务 → 立即执行
- 任务全部完成 → 停止，报告人类
- 有阻塞问题 → 记录问题，切换到可执行任务
```

### 4.3 上下文耗尽处理

```
人类开启新对话：

1. 查看 docs/协调者迭代/ 文件夹
2. 找到最新的协调者记录文件（编号最大）
3. 读取该文件了解当前状态
4. 继续执行或处理阻塞问题
```
