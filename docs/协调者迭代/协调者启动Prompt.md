# 协调者启动 Prompt

请在新窗口中执行以下操作：

---

## 1. 加载协调者环境

### 步骤 1：读取核心配置

```
1. 读取项目根目录的 CLAUDE.md（核心配置）
2. 读取 docs/项目配置.md（项目信息和约束）
3. 读取 docs/requirements/需求迭代文档.md（项目目标）
```

注意：docs/00-通用规范/ 下的详细规范按需加载，启动时不要全部读取。

### 步骤 2：恢复任务状态

```
1. 扫描 docs/协调者迭代/ 文件夹
2. 找到编号最大的协调者记录文件（排除本文件）
3. 如果找到 → 读取该文件恢复任务状态
4. 如果没有找到 → 这是全新开始，需要创建第一个迭代文件
5. 检查 docs/待确认问题/ 是否有未处理的问题
```

### 步骤 3：显示状态摘要并立即执行

```
输出简要状态摘要（不超过5行），然后立即开始执行。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
协调者已就绪 | 阶段：[阶段名]
待办：[下一个任务] | 待确认：[数量]
开始执行：[任务名]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**关键规则**：显示摘要后不要等待人类输入，直接开始执行。

全新开始时：
- 创建 `docs/协调者迭代/协调者001-架构设计阶段.md`
- 按项目配置中的阶段表，从"架构设计"阶段开始

## 2. 协调者权限声明

我已获得系统权限，可以在无需人类确认的情况下：
- 创建/删除/修改任何文件
- 执行 Git 操作（仅提交到本地）
- cmake 构建和运行
- 安装依赖和工具
- 配置开发环境

特殊规则：
- git commit：✅ 可以
- git push：❌ 禁止（由人类手动执行）

### 停止条件

```
停止条件（满足任一才停止）：
1. 任务队列中所有任务全部完成
2. 所有剩余任务都处于阻塞状态且没有可并行的替代任务
3. 遇到必须由人类做出决策的问题且无法绕行
```
