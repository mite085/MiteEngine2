# 编码规范（通用版）

**版本**：v1.0

**用途**：项目代码的通用编码规范，可复用于任何项目

---

## 通用原则

### 核心原则
1. **可读性优先** - 代码首先是给人看的，其次才是给机器运行的
2. **一致性** - 整个代码库保持统一的风格
3. **简洁** - 避免过度工程，用最简单的方案解决问题
4. **命名自文档化** - 好的命名比注释更重要

---

## 命名规范

### 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 头文件 | `.hpp` 或 `.h` | `Renderer.hpp` |
| 实现文件 | `.cpp` | `Renderer.cpp` |
| 模块文件 | 小写下划线 | `sdf_renderer.cpp` |

### 变量命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 普通变量 | 小驼峰 | `renderDevice`, `vertexBuffer` |
| 类成员变量 | 小驼峰 + 下划线后缀 | `renderDevice_` |
| 全局变量 | 小驼峰 + g_ 前缀 | `g_renderContext` |
| 常量 | k 前缀 + 大驼峰 | `kMaxBufferSize` |
| 枚举值 | 大驼峰 | `RenderState::Idle` |

### 函数命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 普通函数 | 大驼峰 | `CreateRenderTarget()` |
| 成员函数 | 大驼峰 | `Initialize()` |
| 回调函数 | On开头 | `OnRenderFrame()` |
| Getter/Setter | Get/Set + 属性名 | `GetWidth()`, `SetHeight()` |

### 类/结构体命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 类 | 大驼峰 | `class RenderDevice` |
| 结构体 | 大驼峰 | `struct Vector3` |
| 接口类 | I前缀 + 大驼峰 | `class IRenderer` |

## 代码结构

### 头文件结构

```cpp
// 1. 头文件保护
#ifndef PROJECT_MODULE_NAME_H
#define PROJECT_MODULE_NAME_H

// 2. 包含标准库
#include <vector>
#include <string>

// 3. 包含项目内部头文件
#include "Core/BaseObject.hpp"

// 4. 前向声明
class IRenderer;

// 5. 命名空间开始
namespace mte {

// 6. 类/函数声明
class Renderer {
public:
    void Initialize();
    void Render();

private:
    std::vector<RenderObject*> objects_;
};

// 7. 命名空间结束
} // namespace mte

#endif // PROJECT_MODULE_NAME_H
```

### 实现文件结构

```cpp
// 1. 对应头文件
#include "Renderer.hpp"

// 2. 包含实现所需的头文件
#include <GLFW/glfw3.h>

// 3. 命名空间开始
namespace mte {

// 4. 函数实现
void Renderer::Initialize() {
    // 实现
}

// 5. 命名空间结束
} // namespace mte
```

## 格式化规范

### 缩进和空格
- 使用 **4空格** 缩进
- 不使用 Tab
- 行长度不超过 **100 字符**
- 函数参数每行一个
- 操作符周围加空格

```cpp
// ✅ 正确
if (condition &&
    anotherCondition) {
    doSomething(param1,
               param2);
}

// ❌ 错误
if(condition&&anotherCondition){
    doSomething(param1,param2);
}
```

### 大括号

```cpp
// ✅ 正确
if (condition) {
    doSomething();
} else {
    doOtherThing();
}

// ❌ 错误
if (condition)
    doSomething();
else
    doOtherThing();
```

## 注释规范

### 原则
- **解释"为什么"，而不是"是什么"**
- 代码本身已经表达的内容不需要注释
- 复杂逻辑、算法、决策需要注释

### 注释风格

```cpp
// 单行注释

/// Doxygen 文档注释（公开API）
/// @brief 初始化渲染器
/// @param config 配置信息
void Initialize(const RenderConfig& config);

/// @brief 内部实现注释（可选）
// 这里使用双缓冲是为了避免画面撕裂
```

## 错误处理

### 原则
- 错误应该尽早检测
- 使用明确的错误码或异常
- 不要忽略错误

```cpp
// ✅ 正确
Result Initialize() {
    if (!CheckRequirements()) {
        return Result::Error("Missing requirements");
    }
    // ...
}

// ❌ 错误
void Initialize() {
    // 静默失败
    CheckRequirements();
}
```

## 性能规范

### 原则
- 先保证正确，再优化
- 避免过早优化
- 关键路径优先优化

### 常见规则
- 避免在循环中分配内存
- 优先使用栈分配
- 减少不必要的拷贝
- 使用移动语义
