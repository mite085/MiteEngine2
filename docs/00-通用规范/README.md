## 复用指南

### 启动新项目时

1. **复制通用规范**
   ```
   复制 docs/00-通用规范/ 到新项目的 docs/
   ```

2. **填写项目配置**
   ```
   复制项目配置模板 → docs/项目配置.md
   填写项目特定信息
   ```

3. **配置 CLAUDE.md**
   ```
   复制 CLAUDE.md 模板 → 项目根目录/
   修改项目引用指向正确的配置文件
   ```

### 复用要点

| 组件 | 复用方式 | 是否修改 |
|------|---------|---------|
| 协调者能力定义 | 直接复制 | ❌ 不需要 |
| Agent Team 角色 | 直接复制 | ❌ 不需要 |
| 通用 Rules | 直接复制 | ❌ 不需要 |
| 项目配置 | 填写模板 | ✅ 必须 |
| 迭代文档 | 新建 | ✅ 必须 |

---

## 核心原则

**通用层（不变）**：
- 协调者能力定义
- 苏格拉底询问法（需求澄清）
- 经验反馈机制（持续优化）
- 长线运营机制（异常处理）
- Agent Team 角色定义
- Agent Prompt 模板
- 编码规范
- 项目脚手架结构
- 工作流程规范
- Rules 和 Skills

**项目层（可配置）**：
- 项目基本信息
- 技术约束
- 需求文档
- 迭代状态
- 项目特定规范
