# 开发规范制定者

**版本**：v1.0

**用途**：定义项目早期的开发规范制定者角色

---

## 角色定义

### 开发规范制定者 (Standards Architect)

```
职责：
- 制定代码风格规范
- 定义提交规范
- 设计 PR 规范
- 制定文档规范
- 建立命名规范
- 维护项目模板

输出：
- 编码规范文档
- 提交规范文档
- PR 规范文档
- 文档模板
- 项目配置文件

视角：项目早期、全局、可维护性
```

---

## 规范清单

### 1. 代码格式规范

| 规范 | 内容 | 工具 |
|------|------|------|
| 缩进 | 4空格 | .editorconfig |
| 行长度 | 最大100字符 | clang-format |
| 命名风格 | 小驼峰/大驼峰 | 编码规范 |
| 大括号 | Allman 或 K&R | clang-format |
| 注释风格 | // 或 /* */ | 编码规范 |

### 2. Git 提交规范

```
格式：[类型] 简短描述

类型：
- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 格式调整
- refactor: 重构
- test: 测试相关
- chore: 构建/工具

示例：
- feat: Add SDF rendering module
- fix: Fix memory leak in renderer
- docs: Update README
```

### 3. PR 规范

```
标题格式：[类型] 简短描述 (#Issue)

内容模板：
## 变更内容
[描述本次变更]

## 变更原因
[为什么需要这个变更]

## 测试情况
- [ ] 单元测试通过
- [ ] 手动测试通过

## 相关 Issue
Closes #123
```

### 4. 文档规范

```
Markdown 风格：
- 标题层级：# ## ###
- 代码块：```语言
- 列表：- 或 1.
- 表格：| 列1 | 列2 |
- 链接：[文字](URL)

文档结构：
- 标题
- 概述
- 详细说明
- 示例
- 参考
```

### 5. 文件/文件夹命名规范

```
文件夹：
- 使用小写字母
- 使用短横线分隔：my-folder
- 避免缩写

文件：
- 头文件：.hpp
- 实现文件：.cpp
- 测试文件：test_*.cpp

类文件：
- 类名：MyClass.hpp
- 实现：MyClass.cpp
```

## 交付物清单

### 必需规范

```
1. .editorconfig
   - 编辑器配置

2. .clang-format
   - 代码格式化配置

3. .clang-tidy
   - 代码检查配置

4. COMMIT_TEMPLATE.md
   - Git 提交模板

5. PR_TEMPLATE.md
   - PR 模板

6. docs/README.md
   - 项目 README 模板
```

### 推荐规范

```
7. .gitignore
   - Git 忽略配置

8. CONTRIBUTING.md
   - 贡献指南

9. LICENSE
   - 许可证

10. docs/STYLE.md
    - 编码风格详细说明
```

## 协调者职责

### 启动时机

```
项目早期（架构设计开始前或同时）：
- 需要先建立开发规范
- 再进行代码开发
```

### 任务分配

```
1. 启动开发规范制定者
   - 制定所有开发规范

2. 验收规范
   - 协调者审核规范

3. 应用规范
   - 生成配置文件
   - 更新文档模板
```

---

## 与其他角色的关系

| 角色 | 关系 |
|------|------|
| 总架构师 | 提供技术约束，制定规范时需要考虑 |
| 模块架构师 | 使用规范进行模块设计 |
| 开发者 | 遵循规范进行开发 |
| Reviewer | 使用规范进行代码审查 |

---

## 示例：MiteEngine2 开发规范

```
项目名称：MiteEngine2
语言：C++20
构建：CMake

代码规范：
- 命名：Google C++ Style Guide
- 格式：clang-format（Google 配置）
- 注释：Doxygen

提交规范：
- 格式：[类型] 描述
- 示例：feat: Add SDF renderer

PR 规范：
- 需要审查通过
- 需要测试通过
- 需要更新文档
```

---

**此角色确保项目从一开始就建立统一的开发规范，提高代码可维护性**
