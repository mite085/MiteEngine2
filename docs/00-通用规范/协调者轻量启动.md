# 协调者轻量启动机制

**版本**：v1.0

**用途**：减少协调者的文档阅读压力，实现快速启动

---

## 1. 核心原则

### 启动分层

```
完整模式：
- 首次启动或需要深入了解项目时
- 读取所有必要文档

轻量模式：
- 后续启动时
- 只读取核心状态文件
- 按需加载详细文档
```

### 按需加载

```
需要时再加载：
- 任务执行中需要参考某规范 → 动态加载
- 不需要时保持简洁
```

---

## 2. 核心文件（轻量启动必读）

### 必读文件（< 5个）

| 文件 | 内容 |
|------|------|
| CLAUDE.md | 核心配置、角色、Rules |
| 协调者迭代/最新文件 | 当前任务、状态 |
| 项目配置.md | 项目基本信息 |

### 按需加载

| 场景 | 需要加载 |
|------|---------|
| 分配架构任务 | 架构师角色体系 |
| 分配开发任务 | AgentPrompt 模板 |
| 遇到问题 | 流转机制 |
| 需要权限 | 权限管理 |

---

## 3. 启动流程优化

### 轻量启动流程

```
1. 读取 CLAUDE.md（约 200 行）
   ↓
2. 扫描迭代文件夹，找到最新文件
   ↓
3. 读取最新迭代文件（通常 < 100 行）
   ↓
4. 显示状态摘要（10 行内）
   ↓
5. 等待人类指令
```

### 任务执行时的文档加载

```
人类指令："开始开发渲染模块"
    ↓
协调者分析需要：
- 模块架构师的职责
- 开发者的输入要求
    ↓
按需加载：
- 读取架构师角色体系（只读相关章节）
- 读取 AgentPrompt 模板（只读开发者部分）
    ↓
开始任务
```

---

## 4. 状态摘要格式

### 轻量摘要（约 10 行）

```
当前状态：架构设计
协调者：001 | 创建于 2026-03-08
待办：1. 架构设计 2. 开发规范制定
阻塞：无
进度：Team 建设完成
```

### 不需要读取

- 完整的通用规范（20+ 文档）
- 历史迭代文档
- 不相关的配置

---

## 5. 文档维护

### 迭代文件精简原则

```
每个迭代文件应该：
- 包含当前状态（< 100 行）
- 包含待办任务
- 包含关键决策摘要
- 链接到详细文档（不直接读取）
```

### 摘要模板

```markdown
# 协调者工作记录

**编号**：001
**阶段**：架构设计
**创建**：2026-03-08

## 状态
- 待办：架构设计
- 阻塞：无

## 决策（详细见 xxxx.md）
- 决策1：xxx
- 决策2：xxx

## 进度
- Team 建设：完成
- 架构设计：待开始
```

---

## 6. 人类指令优化

### 人类可以提供上下文

```
好的启动方式：
"开始架构设计，参考需求迭代文档.md"

协调者只需要：
- 读取需求文档
- 开始执行任务
```

---

## 7. 总结

**核心思路**：
- 启动时只读核心状态文件（< 5 个）
- 任务执行时按需加载详细规范
- 保持迭代文件精简
- 人类指令可附带上下文

**这减少了协调者的初始阅读压力，同时保留了按需获取详细信息的能力**
