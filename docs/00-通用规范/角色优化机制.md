# 角色优化机制

**版本**：v1.0

**用途**：定义协调者和项目经理如何迭代优化 Agent 团队组织架构

---

## 1. 优化权力

### 授权范围

```
协调者和项目经理有权：
1. 识别需要优化的角色/流程问题
2. 提出角色调整方案
3. 更新通用规范中的角色定义
4. 创建新角色或合并/删除现有角色
5. 调整职责边界
```

### 不需要人类审批的情况

```
日常优化：
- 调整 Agent 职责边界
- 添加子角色（如添加"安全审查员"子角色）
- 优化流程细节
- 更新文档描述
```

### 需要人类审批的情况

```
重大变更：
- 创建全新类型的角色
- 删除现有角色
- 改变核心流程
- 改变权限等级
```

---

## 2. 优化触发条件

### 可以触发优化的情况

| 情况 | 示例 |
|------|------|
| 发现职责重叠 | 两个人做同样的事 |
| 发现职责空白 | 没人负责某件事 |
| 发现流程问题 | 流程导致效率低下 |
| 发现角色多余 | 某个角色从未被使用 |
| 发现需要新角色 | 新需求没人负责 |

---

## 3. 优化流程

### 流程图

```
发现问题
    ↓
分析问题
    ↓
    ├─→ 日常优化 → 协调者/项目经理决定 → 更新规范
    │
    └─→ 重大变更 → 向人类提案 → 人类审批 → 执行
```

### 优化步骤

```
1. 记录问题
   - 什么问题
   - 什么场景发现
   - 影响是什么

2. 分析方案
   - 可能的解决方案
   - 优缺点对比
   - 建议方案

3. 决策
   - 日常优化：协调者/项目经理直接决定
   - 重大变更：向人类提案

4. 执行
   - 更新相关文档
   - 记录变更
   - 通知相关 Agent
```

---

## 4. 优化记录

### 记录格式

```markdown
## 角色优化记录

### [日期] - [优化主题]

**问题**：
[描述问题]

**分析**：
[分析过程]

**决定**：
[决定内容]

**变更**：
- 修改了 xxx 文档
- 新增/删除了 xxx 角色
```

### 示例

```markdown
## 角色优化记录

### 2026-03-15 - 添加安全审查员

**问题**：
Reviewer 发现安全漏洞但缺乏专门的安全知识

**分析**：
- 安全问题需要专门知识
- 可以创建"安全审查员"子角色
- 或者让 Reviewer 学习安全知识

**决定**：
添加"安全审查员"作为 Reviewer 的子角色

**变更**：
- 在 AgentTeam工作规范.md 添加子角色
- 更新 AgentPrompt模板.md 添加安全审查模板
```

## 6. 迭代原则

### 优化原则

```
1. 最小变更：只改必要的部分
2. 保持兼容：新角色尽量兼容现有流程
3. 记录决策：所有变更都要记录原因
4. 可回滚：变更要可追溯，必要时可回滚
5. 及时同步：变更后更新所有相关 Agent
```

### 不要过度优化

```
避免：
- 为了优化而优化
- 创建过多细分角色
- 过度工程化
- 频繁变更
```

---

## 7. 协调者/项目经理职责

### 协调者职责

```
1. 监控整体运行
2. 识别优化机会
3. 做出日常优化决策
4. 向人类提案重大变更
5. 更新相关文档
```

### 项目经理职责

```
1. 日常观察问题
2. 收集优化建议
3. 向协调者报告问题
4. 协助执行优化
```

---

**此机制确保 Agent 团队能够持续优化，而不需要人类事必躬亲**
