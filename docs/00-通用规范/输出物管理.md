## 2. 各 Agent 产出物位置

### 总架构师

```
产出：
- 总体架构文档
- 模块划分方案
- 技术选型报告

存放位置：
docs/01-架构/
├── 总架构文档.md          # 总体架构设计
├── 模块划分方案.md        # 模块划分
└── 技术选型报告.md        # 技术选型决策
```

### 接口人

```
产出：
- 模块接口规范
- 依赖关系图
- 集成测试方案

存放位置：
docs/01-架构/
├── 接口规范.md            # 模块接口定义
└── 依赖关系图.md          # 模块依赖
```

### 模块架构师

```
产出：
- 模块架构文档
- 模块接口定义

存放位置：
docs/01-架构/模块架构/
├── 渲染模块架构.md
├── 物理模块架构.md
├── ECS模块架构.md
└── ...
```

### 开发规范制定者

```
产出：
- 编码规范文档
- 配置文件（.editorconfig, .clang-format）

存放位置：
docs/02-规范/
├── 编码规范.md
└── 开发规范.md

配置文件：
根目录/
├── .editorconfig
└── .clang-format
```

### 开发者

```
产出：
- 源代码
- 单元测试
- 代码变更说明

存放位置：
src/                    # 源代码
include/                # 头文件
tests/unit/             # 单元测试
```

### 测试

```
产出：
- 测试代码
- 测试报告
- 测试用例文档

存放位置：
tests/                  # 测试代码
docs/04-报告/测试报告/  # 测试报告
```

### Reviewer

```
产出：
- 审查报告

存放位置：
docs/04-报告/审查报告/
├── PR001-审查报告.md
├── PR002-审查报告.md
└── ...
```

### 项目经理

```
产出：
- 进度报告
- 任务状态更新

存放位置：
docs/04-报告/进度报告/
├── 2026-03-周报.md
└── ...
```

## 3. 文档命名规范

### 架构文档

```
格式：[模块名]-架构-[版本].md

示例：
- 渲染模块-架构-v1.0.md
- 物理模块-架构-v1.0.md
```

### 审查报告

```
格式：PR[编号]-[标题]-审查报告.md

示例：
- PR001-SDF渲染模块-审查报告.md
- PR002-修复内存泄漏-审查报告.md
```

### 进度报告

```
格式：YYYY-MM-DD-[类型]-报告.md

示例：
- 2026-03-09-周报.md
- 2026-03-09-月报.md
```

## 4. 版本管理

### 文档版本

```
架构文档版本控制：
- 每次重大变更创建新版本
- 旧版本保留作为历史参考
- 使用 v1.0, v1.1, v2.0 格式
```

### Git 提交规范

```
文档提交：
- feat: 添加 xxx 架构文档
- docs: 更新 xxx 规范
- refactor: 重构 xxx 模块设计
```

## 6. 清理规则

### 需要清理的内容

```
- 构建产物（build/）
- 临时文件（*.tmp, *.log）
- 编译缓存（.cmake/, compile_commands.json）
- IDE 配置（.vs/, .idea/）
```

### 清理频率

```
每次提交前：
- 检查是否有不应该提交的文件
- 清理临时文件

每周：
- 清理 build/ 目录
- 清理编译缓存
```
